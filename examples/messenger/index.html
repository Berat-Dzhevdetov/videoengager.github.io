Limitations:

  • Web Messenger must be loaded. If the Web Messenger hasn't fully loaded, messages cannot be sent.
  • Cannot End Calls from the Visitor's Side: Currently, we lack the implementation to define and manage the visitor's ID and the state of the smartvideo. As a result, it's not possible for visitors to initiate a call and then cancel it. The agent is required to answer and subsequently end the call.
  • Messenger must be opened and loaded in order to send messages and custom attributes.
  • We cannot terminate the interaction from visitor side: We use the same interaction each time and we cannot remove the interaction we placed
